# Core: Certificate Import (Resource)

Imports or replaces a DSM certificate from certificate files already stored on the NAS.

The file paths must be Synology File Station paths rooted at a shared folder, such as
`/docker/certbot/letsencrypt/live/synology.example.com/cert.pem`.



<!-- schema generated by tfplugindocs -->
## Schema

### Required

- `certificate_path` (String) NAS path to the PEM certificate file.
- `description` (String) DSM certificate description.
- `intermediate_certificate_path` (String) NAS path to the PEM intermediate certificate chain file.
- `private_key_path` (String) NAS path to the PEM private key file.

### Optional

- `as_default` (Boolean) Whether DSM should set this certificate as the default certificate. Default: `false`.

### Read-Only

- `common_name` (String) Certificate subject common name.
- `id` (String) DSM certificate ID.
- `is_broken` (Boolean) Whether DSM marks this certificate as broken.
- `is_default` (Boolean) Whether DSM marks this certificate as the default certificate.
- `subject_alternative_names` (List of String) Certificate subject alternative names.
- `valid_till` (String) DSM certificate expiry timestamp string.

# Core: Share Nfs Privilege (Resource)

Manages the complete DSM NFS export rule set for a shared folder.

The Synology DSM API exposes NFS privileges per shared folder as a full ordered rule list.
This resource owns that full list for one share. Applying the resource replaces the share's
current DSM NFS rule set with the Terraform-managed list.

## Example Usage

```terraform
resource "synology_core_share_nfs_privilege" "media" {
  share_name = "media"

  rules {
    client      = "10.1.0.0/24"
    privilege   = "rw"
    root_squash = "root"
    async       = true
    crossmnt    = true
    insecure    = true

    security_flavor {
      sys = true
    }
  }

  rules {
    client      = "10.1.0.42"
    privilege   = "ro"
    root_squash = "root"

    security_flavor {
      kerberos_integrity = true
    }
  }
}
```

### Read-Only

- `id` (String) Terraform resource identifier. Matches `share_name`.

<a id="nestedatt--rules"></a>
### Nested Schema for `rules`

Required:

- `client` (String) Client host, IP, CIDR, or wildcard accepted by DSM for this NFS rule.
- `root_squash` (String) DSM API root squash value for the client rule, using the raw keyword DSM returns for the export, for example `root`.

Optional:

- `async` (Boolean) Whether to allow asynchronous writes for the client rule.
- `crossmnt` (Boolean) Whether the client may access mounted subfolders beneath the share export.
- `insecure` (Boolean) Whether DSM should allow client connections from non-privileged ports.
- `privilege` (String) DSM NFS access privilege for the client. Supported values are `ro` and `rw`.
- `security_flavor` (Attributes) Security flavors accepted for this client rule. If omitted, the resource defaults to `sys = true`. (see [below for nested schema](#nestedatt--rules--security_flavor))

<a id="nestedatt--rules--security_flavor"></a>
### Nested Schema for `rules.security_flavor`

Optional:

- `kerberos` (Boolean) Allow Kerberos (`krb5`).
- `kerberos_integrity` (Boolean) Allow Kerberos integrity (`krb5i`).
- `kerberos_privacy` (Boolean) Allow Kerberos privacy (`krb5p`).
- `sys` (Boolean) Allow AUTH_SYS (`sys`).

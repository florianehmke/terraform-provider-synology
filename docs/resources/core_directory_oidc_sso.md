# Core: Directory Oidc Sso (Resource)

Manages the DSM OpenID Connect SSO client configuration.

This resource models the singleton DSM OIDC SSO client under Control Panel -> Domain/LDAP -> SSO Client.
It manages the OIDC client settings, whether the OIDC profile is enabled, and whether DSM should select SSO by default on the login page.



<!-- schema generated by tfplugindocs -->
## Schema

### Required

- `client_id` (String) OIDC client ID.
- `client_secret` (String, Sensitive) OIDC client secret.
- `name` (String) Display name shown in DSM for the OIDC SSO client.
- `redirect_uri` (String) DSM redirect URI used for the OIDC SSO client.
- `username_claim` (String) OIDC claim DSM should use as the username.
- `wellknown_url` (String) OIDC discovery document URL.

### Optional

- `allow_local_user` (Boolean) Whether DSM should allow local users for the OIDC SSO client. Default: `true`.
- `default_login` (Boolean) Whether DSM should select SSO by default on the login page. Default: `false`.
- `enabled` (Boolean) Whether DSM should enable the OIDC SSO profile. Default: `true`.
- `scope` (String) Authorization scope string. Default: `openid profile email`.

### Read-Only

- `authorization_endpoint` (String) Resolved authorization endpoint from the well-known URL.
- `id` (String) Stable singleton resource ID.
- `token_endpoint` (String) Resolved token endpoint from the well-known URL.

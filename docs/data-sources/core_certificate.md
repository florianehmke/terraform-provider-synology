# Core: Certificate (Data Source)

Looks up a DSM certificate by ID or domain.

When `domain` is used, the provider selects the best non-broken matching certificate,
preferring the DSM default certificate and then the latest expiry time. Wildcard SAN/CN matching
is supported for single-label wildcards such as `*.synology.example.com`.

## Schema

### Optional

- `certificate_id` (String) Exact DSM certificate ID to look up.
- `domain` (String) Domain to match against certificate CN or SAN entries.

### Read-Only

- `common_name` (String) Certificate subject common name.
- `description` (String) DSM certificate description.
- `id` (String) Selected DSM certificate ID.
- `is_broken` (Boolean) Whether DSM marks this certificate as broken.
- `is_default` (Boolean) Whether this is the current DSM default certificate.
- `subject_alternative_names` (List of String) Certificate subject alternative names.
- `valid_till` (String) DSM certificate expiry timestamp string.
